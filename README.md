# Arogent — Confidence in Every Screening

A decision-support MVP for community diabetes screening, built for India's Ayushman Arogya Mandirs / ASHA network.

**Arogent Verify** estimates a Screening Confidence Score — using four independent consistency signals — before **Arogent Risk** generates any diabetes risk prediction. The confidence gate is enforced in the backend service layer itself (`app/risk/service.py`'s `ConfidenceGateError`), not just documented as a convention: Arogent Risk structurally cannot run on a screening that didn't reach HIGH confidence.

This is a **decision-support tool**. It does not diagnose disease, does not claim to verify or prove a screening is genuine, and never replaces clinical judgement — a clinician always makes the final call.

---

## Data Disclosure

All AI components (Arogent Verify's Isolation Forest, Arogent Risk's diabetes model) are trained on a **synthetic dataset** of simulated community screening records generated by `backend/app/ai/synthetic_data.py`. No real patient, ASHA, or government health data is used anywhere in this project.

## The Full Pipeline

```
Login -> Register Patient -> Screening -> Arogent Verify
                                              |
                                    Confidence == HIGH?
                                         |           |
                                        Yes          No
                                         |           |
                                   Arogent Risk   Repeat Screening /
                                         |        Escalate for PHC Review
                                    risk_level == HIGH?
                                         |
                                        Yes
                                         |
                                    Referral Generated
                                         |
                                         v
                                   Result Screen -> District Dashboard
```

## Quick Start

```bash
# Backend
cd backend
python3 -m venv venv
./venv/bin/pip install -r requirements.txt
cp .env.example .env   # then edit SECRET_KEY before anything beyond local dev
./venv/bin/python -m app.ai.train_anomaly_model   # trains Arogent Verify's Behaviour signal
./venv/bin/python -m app.ai.train_risk_model      # trains Arogent Risk
./venv/bin/uvicorn app.main:app --reload

# Frontend (separate terminal)
cd frontend
npm install
npm run dev
```

Full walkthrough: [`docs/INSTALLATION.md`](docs/INSTALLATION.md).

## What's Deliberately Not Built

Documented explicitly rather than left ambiguous — see each area's own doc for detail:

- **Anaemia / hypertension / TB screening pathways** — diabetes only, fully working (proposal Section 4)
- **Recent Screenings dashboard table** — no backend endpoint exists for a district-wide recent-screenings list yet (see `frontend/src/components/dashboard/RecentScreeningsTable.tsx`)
- **Row-level data scoping** (e.g. an ASHA seeing only their own village's patients) — auth confirms *who* you are; nothing yet scopes *what* you can see (see `docs/SECURITY.md`)
- **Token revocation / refresh tokens** — a compromised JWT remains valid until its 12-hour natural expiry
- **Real government/PHC data integration (ABHA, etc.)** — synthetic data throughout, by design
